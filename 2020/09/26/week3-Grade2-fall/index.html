1. 红黑树

30张图彻底理解红黑树

1.1 红黑树定义

  • 基本限制：
    • 满足二叉搜索树；
    • 每个节点要么是黑色的，要么是红色的；
    • 所有的外部节点都是黑色的（每个叶子结点【NIL】是黑色）；
    • 颜色限制：红色结点没有红色的子节点（每个红色结点的两个子节点一定都是黑色的）；
    • 黑色高度限制：每个从给定结点u到外部节点的路径的黑色长度相等。（任意一结点到每个叶子结点到路径都包含数量相同的黑结点）
      • 推论：如果一个结点存在黑子节点，那么该结点肯定有两个子节点；
    • 根结点是黑色的；
  • ARB tree (Almost-red-black tree)
    • 根结点是红色的；
  • 递归定义
    • 约定高度为h的红黑树为RBh，高度为h的Almost-red-black Tree为ARBh
    • 一个二叉树是ARBh，那么
      • 根结点是红色的；
      • 左右子树都是RBh-1
    • 一个二叉树是RBh，那么
      • 根结点是黑色的；
      • 左右子树是RBh-1或ARBh

1.2 红黑树的插入

  • ```c++
    RBtree rbtInset(RBtree oldRBtree, Element newElement){
    InsReturn ans = rbtIns(oldRBtree, newNode);
    if(ans.newTree.color != black){
    ans.newTree.color = black; //黑高度增加的途径。
    }
    return ans.newTree;
    }
    
- **插入红色结点不会影响黑色高度限制**；【每次插入都不改变黑高度】

- 簇：一个黑色结点和所有可从该黑色结点经由非黑色边达到的红色结点组成的集合，称为**簇**。

- 冲突簇（Critical cluster）：起因于3或4个红色结点的颜色冲突。如果存在一个从根结点开始到达某结点的通路的长度大于1，那么这个簇称为**冲突簇**。因为一个簇中的所有边都是非黑边，那么存在一条长度为2的通路说明簇中一个红色结点与另一个红色结点之间有一条边。

- 颜色漂移（Color flip）：冲突簇的根结点和它的子结点变换颜色，当没有新的冲突簇出现时，插入结束。（教材原话：We perform a *color flip* with the root of the cluster, call it *r*, and its two children.）对于三个结点的冲突簇情况不能使用颜色漂移。【三个结点的情况使得中间那个结点变为黑色】

  - ```c++
    void rb_tree_main::colorFlip(rb_tree * old_tree){
      //修改根结点的颜色
      if(old_tree->color == color_black){
        old_tree->color = color_red;
      } else {
        old_tree->color = color_black;
      }
      
      //修改根的左子结点的颜色
      if(old_tree->left->color == color_black){
        old_tree->left->color = color_red;
      } else {
        old_tree->left->color = color_black;
      }
      
      //修改根的右子节点的颜色
      if(old_tree->right->color == color_clack){
        old_tree->right->color = color_red;
      } else {
        old_tree->right->color = color_black;
      }
    }

1.3 修正红黑树（RepairLeft 和 RepairRight）

  • ok：不需要修正；

    • ```c++
      if(ansLeft.status == status.ok){
      ans.new_tree = old_tree;
      ans.status = status.ok;
      return ans
      }
      old_tree->left = ansLeft.new_tree;
      
- rbr：不要调整；要将返回值存入相应的field。

  - ```c++
    if(ansLeft.status == status.rbr){
      ans.new_tree = old_tree;
      ans.status = status.ok;
    }

  • brb：如果父结点是黑色的，不修正；否则需要向上调整（若当前不调整，返回的状态为rrb）

    • ```c++
      if(ansLeft.status == status.brb){
      ans.new_tree = old_tree;
      if(old_tree->color == color_black){
      ans.status = status.ok;
      } else {
      ans.status = status_rrb;
      }
      }
      
- rrb：【**当前的父结点的颜色一定为黑色**】根据oldTree的右（左）子节点的情况进行调整（分别对应4-node和3-node两种颜色冲突的情况）；

  - ```c++
    if(ansLeft.status == status.rrb){
     if(old_tree->right->color == color_red){
       colorFlip(old_tree);
       ans.new_tree = old_tree;
       ans.status = status.brb;
     } else {
       ans.new_tree = rebalLeft(old_tree, ansLeft.status);
       ans.status = rbr;
     }
    }

  • brr：和rrb的处理情况类似；

  • 所有的处理都不会对黑高度造成影响；

1.4 红黑树的删除

  • 只有一个子节点的结点可以直接删除；
  • 保持颜色限制，调整高度。
    • 删除红色结点，黑高度不变，没有任何影响；
    • 删除黑色结点，该结点所在的子树的黑高度减少了1，需要进行修正。并且这种修正可能导致更高的子树的黑高度降低，需要继续修正。
  • 删除时，将该结点与它在中序遍历之后的后一个结点互换。删除互换后的该结点。将该结点的右结点代替被删除的集合。

6. Division

  • 将被除数（remainder）的左边补上n位的符号位，除数左边补上n位的0，每一次将除数减被除数，再右移除数一位，若差大于等于0，则在此处的商置为1，将商左移一位。（原始版本）
    • （remainder和quotient合在一起）依然是用division减remainder（前面的n位），如果是异号则用加法。
  • 余数一定和被除数同号（用除数不断去逼近被除数，但是不能越过它）
  • 当除数和被除数异号的时候，每次操作的减法变为加法。
  • 若每次的加法或减法溢出，则做逆操作使得被除数恢复（recover）。
  • 如果被除数和除数的异号的，我们需要对商修正——取反加一。
  • 将错就错版本：
    • 符号扩展n位，然后存储在余数-商寄存器；
    • 如果被除数和除数有相同的符号做减法，否则做加法：
      • 如果余数和除数有相同的符号，Qn为1，反之为0。
    • 如果余数和除数有相同的符号，那么 Ri+1 = 2Ri - Y ，否则 Ri+1 = 2Ri + Y （注：有相同符号说明不需要补上之前错减的操作，继续-Y就可以；若不同说明需要减去之前的操作，因为前面已经错减了，左移*2之后，加上一个Y就可以）
      • 如果新的余数和除数有相同的符号，将商置为1，否则置为0。
    • 重复。
