算法设计与分析

0. 尾递归

我们常常用尾递归的方式将递归转化为迭代。

0.0. 回顾 SICP 中 Scheme 的尾递归

因为函数式语言的最大特征之一是无法赋值，因此无法使用过程式语言中的 for 或者 while 循环来实现迭代。为了减小开销使得递归的效率更高，Scheme 引入了尾递归。(调用之后释放空间，与传统递归不同)

(define (fact n)
  (define (fact-tail n result)
    (if (<= n 1)
        result
        (fact-tail (- n 1) (* n result))))
  (fact-tail n 1))

Scheme 中的递归会使用一个辅助函数。每调用一次辅助函数之后，函数不需要在返回到调用者的函数，而是直接带着 result 调用后续的函数。

0.1. 尾递归的基本概念

  • 尾递归：一个递归函数仅仅在运算的最后才递归调用自身，并且调用后直接返回。

  • 尾递归转换为迭代的方法：

    • 将每个参数转换为变量声明；
    • 将函数体转换为一个循环体，并且对于基本情况直接返回相应的结果；递归调用时，首先计算新的参数并赋予相关的变量，后面跟上 continue 的语句。

  • e.g.

    Euclid_recursion(int m, int n){
	if(n == 0) return m;
    else {
        int rem = m % n;
        return Euclid_Recursion(n, rem);
    }
}

1. 排序问题和排序算法-1

1. 1. 插入排序（Insertion Sort)

1.1.1. 插入排序的基本内涵和实现

  • 基本情况：n == 1时，不需要排序。

  • 递归情况：n > 1时，分成两组——任意数 v 和其余的n-1个数；然后递归地排好剩下的n-1个数，再将v插入到已经排序完成的数列的适当位置，得到排序完成的数组。

  • 如何插入？(下面的Insert_Abs就是插入算法)

    public static int[] Insert_Abs(int v, int[] list){
	if(list.length() == 0) return v;
	else if (v <= Head(list)) return {v} + list;
	else return Head(list) + Insert_Abs(v, Tail(list));
}

  • 实现

    public static int[] insert(int[] A){
    int i, j;
    int len = A.length;
    int target;

    for(i=1; i<len-1; i++){
        j = i;
        target = A[i];					//等价于ppt中推导时所用到的v
        while(j>0 && target<A[j-1]){	//一旦target比前面的元素大，那么就插入它的后面
            A[j] = A[--j];				//否则将元素后移一位，空出位置给target。
        }								//这个算法的实现和课上所讲的略有不同
        A[j] = target;					//这个算法是从前向后插入，课上是从后向前
    }
    return A;
}
1.1.2. 插入排序 Insert 函数的循环不变式
  • 初始情况下，从start+1到end是排好序的；
  • 在经过一次操作之后，从start到end是排好序；
  • 循环一定是可以终止在 i = len-1的；
1.1.3. 最坏情况分析

  • 最坏情况：逆序

  • 时间复杂度：Ο(n2)

  • 平均情况：对于每一个 i ，对应会有 j 个比较次数（内循环）。

    求和之后，仍是 Ο(n2)。

1.2. 快速排序

1.2.1 基本思路和实现

  • 按照某个数进行划分，小于这个数的作为一部分，大于这个数的作为另一部分。

  • 采用分划算法 Partition(v, list) 将list分划成两个list：smallerList中的值都小于v，biggerList中都大于等于v。

  • 有关实现：

    • Partition 函数分割后的值仍然存放在这个区域。
    • QuickSort 之后得到的数据仍然存放在相同的区域。

  • 以下是“挖坑法”：（均在数组的原地址空间内进行改动）

    public static void quickSort(int[] arr, int start, int end){
  if(end <= start) return;
    int pivotIndex = partition(arr, start, end);
    quickSort(arr, start, pivotIndex-1);
    quickSort(arr, pivotIndex+1, end);
}

public static int partition(int[] arr, int start, int end){
    int pivot = arr[start];
    int left = start;
    int right = end;
    int index = start; //挖坑的位置

    while (left <= right){
        //从右边向左边找到比标记更小的，将它转移到标记的左边
        while (left <= right){
            if(arr[right] < pivot){
                arr[left] = arr[right];
                index = right;
                left++;
                break;
            }
            right--;
        }

        //从左向右找到比标记更大的，将它换转移到标记的右边，若上面发生了更换，那么这里也一定是把Pivot填写在上一个循环挖出的空间中；
        while (left <= right) {
            if (arr[left] > pivot){
                arr[right] = arr[left];
                index = left;
                right--;
                break;
            }
            left++;
        }
    }
	//补上漏洞
    arr[index] = pivot;
    return index;
}
    • “指针交换法”：先把pivot以外的元素排好，再把pivot插入到其中相应的位置。
    public static void quickSort(int[] arr, int start, int end){
    if(start >= end) return;
    int pivotIndex = partition(arr, start, end);
    quickSort(arr, start, pivotIndex-1);
    quickSort(arr, pivotIndex+1, end);
}

public static int partition(int[] arr, int start, int end){
    int pivot = arr[start];
    int left = start;
    int right = end;

    while (left < right) {
        //控制right指针比较并左移
        while (left < right && arr[right]>pivot){
        right--;
    	}

        //控制right指针比较并右移
        while (left < right && arr[left] <= pivot){
            left++;
        }

        //交换left和right指向的元素
    	if(arr[left]>arr[right]){
            int p = arr[left];
            arr[left] = arr[right];
            arr[right] = p;
        }
    }
    
    //pivot和指针重合点交换，大概是中点的位置
    int p = arr[left];
    arr[left] = arr[start];
    arr[start] = p;

    return left;
}
    • 可以用“栈”重写非递归形式的快速排序：
    public static void quickSort(int[] arr, int start, int end){
    Stack<Map<String, Integer>> quickSortStack = new Stack<Map<String, Integer>>();
    Map<String, Integer> rootParam = new HashMap<String, Integer>();
    rootParam.put("start", start);
    rootParam.put("end", end);
    quickSortStack.push(rootParam);
  
    while (!quickSortStack.isEmpty()) {
        Map<String, Integer> param = quickSortStack.pop();
        int pivotIndex = partition(arr, param.get("start"), param.get("end"));
        if(param.get("start") < pivotIndex-1){
            Map<String, Integer> leftParam = new HashMap<String, Integer>();
            leftParam.put("start", param.get("start"));
            leftParam.put("end", pivotIndex-1);
            quickSortStack.push(leftParam);
        }
  
        if(pivotIndex+1 < param.get("end")) {
            Map<String, Integer> rightParam = new HashMap<String, Integer>();
            rightParam.put("start", pivotIndex+1);
            rightParam.put("end", param.get("end"));
            quickSortStack.push(rightParam);
        }
    }
}

计算机组织与结构

1. Introduction 前瞻

1.1. 结构与组织 Architecture & Organization

1.1.1. Architecture 结构
  • 对程序员可见（visible to programmer）；
  • 对程序的逻辑加载有直接影响；
1.1.2 Organization 组织
  • 对程序员不可见（encapuslated to programmer）；
  • 操作单元和内部链接；

1.2. 冯诺依曼架构

1.2.1. 组成部分
  • 存储器：地址和存储的内容；
  • 处理单元：执行信息的实际处理；
  • 控制单元：指挥信息的处理；
  • 输入设备：将信息送入计算机中；
  • 输出设备：将处理结果以某种形式显示在计算机外；

1.3. 摩尔定律

单个芯片上的晶体管数量每过18个月就会翻一倍。

1.4. CPU表现（CPU Performance）

1.3.1 系统时钟
  • 时钟频率（Clock rate）：计算机在进行基本操作时，一秒内的时钟周期频数。
  • 时钟周期（Clock cycle）：CPU的一次电磁脉冲；
  • 周期时间（Cycle time）：脉冲之间的时间；
1.3.2 指令加载

  • CPI（Cycles Per Instruction）：一条指令所经历的时钟周期数；

  • 总CPI的计算公式：

  • 处理时间

    p：从处理器到存储器之间传输消耗的时钟周期；

  • Million Instructions Per Second (MIPS)

  • Million Floating Point Operations Per Second (MFLOPS)

1.3.3 评价标准（Benchmarks）

  • 评测系统的表现；

  • 平均结果：

    • 算术平均值：结果之和除以总数，得到平均的结果；
    • 调和平均值：一次除以结果得到单次结果的倒数，对其求和再去除总次数得到调和平均值；

2. 计算机的顶层视角

2.1. 计算机组成元素

  • 数据和指令存储在一个可读可写存储器；
  • 存储器可寻址；
  • 指令的加载按一定顺序进行；

2.1.1. 存储器

  • Memory Wall：数据从主存到CPU之间的传输速度的发展速度难以跟得上CPU的发展速度。
  • 解决方案：在主存和CPU之间添加一个cache来减少主存的高频访问和数据传送率；提高一次获取的信息量；
2.1.1.1. 约束条件
  • 容量：越大越好；
  • 速度：跟得上处理器；
  • 成本：相比其他组件可以接受；
2.1.1.2. 约束条件之间的关系
  • 访问时间越短，单位比特的成本越高；
2.1.1.3. 解决措施
  • 存储器层次结构（金字塔）；

2.1.2. I/O

  • 在存储器、CPU和外部资源之间交换数据的模块；
2.1.2.1. Problem
  • I/O 表现不能够跟得上 CPU 速度的增长；
  • 解决方案：缓冲区；

2.1.3. CPU

2.1.3.1. Problem
  • CPU 在等待 I/O 设备时需要等待；
  • 解决方案：
    • 中断：一种被其他模块打断并可能影响正常处理顺序时的机制；
  • 中断检测
2.1.3.2. 多重中断（分优先级嵌套中断）
  • 还有一种执行完一个中断再执行下一个的中断；
2.1.3.3. Theoretical maximum performance

  • 随着技术的发展，出现了流水线、高速缓存、多核多线程等等使得计算机性能表现突飞猛进；
  • 时钟频率也在不断上涨，但是上涨到了一定数值就停滞了。如果时钟频率过高，单条指令的处理时间会缩短，处理能力会下降。

2.1.4. 总线 Bus

  • 连接多个设备的通路——总线；
2.1.4.1. 数据传输类型
  • 数据线：在系统模块之间传输数据；
  • 地址线：指定数据线上和I/O端口上的数据的来源或者目的地的地址；
  • 控制线：控制【访问】、【数据的使用】和【地址线的使用】；
  • 数据线和地址线上的内容可以复用；控制线上的内容不能复用；
