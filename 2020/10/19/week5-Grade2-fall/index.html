计算机组织与结构

1. 内存

  • 能存；
  • 能初始化；
  • 能够读出；

1.1 RAM

  • 读写都是简单且快速的；
  • 但是有易失性（断电后不保存）；
1.1.1 DRAM
  • 动态指的是：电容会有漏电的现象，需要刷新存储器中的内容，因此读写不能同时进行；
  • 有点：成本低，集成度高；
  • 缺点：不稳定，速度慢；
  • 用作主存储器；
1.1.2 SRAM
  • 缺点：集成度低，占空间；
  • 用作Cache；

1.2 ROM

  • 只读存储器；
  • 不需要能量来维持存储的信息；非易失；
  • 只能写一次；通过出场设置来进行写；
  • 安全性更好；

1.3 PROM

  • 大批量生产时，效率没有ROM高；
  • 可编程ROM；
  • 更加灵活和方便——个人使用；

1.4 Read-Mostly Memory

  • 读操作远频繁于写操作；
  • 非易失性存储；
1.4.1 EPROM
  • 可擦除可编程只读存储器；
  • 相比于PROM更贵，但是可以多次使用；
  • 一次性擦除一整个芯片，擦除时间20分钟；
1.4.2 EEPROM
  • 电可擦除可编程只读存储器；
  • 每次重写一个字节；
  • 比EPROM更贵，但也有更低的集成度；
  • 每次写操作需要几百微秒，写操作很慢；

1.5 Flash Memory

  • 闪存，用于U盘。
  • 功能性和稳定性介于EPROM和EEPROM之间；
  • 和EPROM一样的高集成度；
  • 可以在块级擦除，但字节不行；

1.6 Refreshing

集中式刷新
  • 刷新的过程中不能对存储器进行操作；
  • 停止读写操作，集中刷新
非集中式刷新
  • 每次存储周期都刷新——读写周期会延长；
异步刷新
  • 避开现在正在使用的一行，若有一行一直都没有刷新，快到64ms都要强制刷新；

2.4 Elements of Cache Design

  • Cache Size 越大越好？

    • 不是，变好的幅度会越来越小。“局部性”，并不会牵扯到太大的范围，并且块越大块内寻找的成本就会越高。块太大了之后，带来的数据是局部性比较差的、比较老的数据。

  • Mapping Function

    • Cache line
    • 直接映射 Direct Mapping
      • 地址 = Tag + 行号 + 块内地址；
        • 先根据行号找到Cache对应的行，然后比对tag，若不相同则miss，否则根据块内地址选择对应的字节。若miss，根据tag+行号找到主存储器中的内容（低几位）相同，将内容放入cache并且重写tag，将放入内容（根据块内地址查找）送入cpu。
      • 每个块只能存入一个地方。查找很快。映射的时候也很快，每个块只有一个地方可以去。换的也快。
      • 缺点：很容易引起“抖动（Thrashing）”。Cache小的时候容易引起抖动。换一个（很容易出现这个情况）就会换一整个块。
    • 关联映射 Associative Mapping
      • 地址 = Tag + 块内地址；
      • 根据tag寻找（遍历）
      • 避免了抖动，但是实现相对复杂，寻找的成本比较高。
    • 直接映射适合大容量cache，关联映射适合小的；
      • 大容量cache不容易出现抖动。
      • 直接映射查找快，大cache影响小；关联映射遍历查找，小cache效率更高。
    • 组关联 Set Associative Mapping
      • 行的大小和块的大小一样大。

替换算法

  • LRU 最近最少用算法；
  • FIFO 先进先出；
  • LFU 使用频率最少；
  • Random 随机。

写策略

  • Write Through

    • 替换的时候同时写主存和Cache；
    • 一改就要访问主存；
    • 提高了cache和主存的一致性；

  • Write Back

    • 当块被替换时，只修改Cache。；
    • 需要一个脏位来表示这一个块是否被调整过；如果是1就写回去，如果是0就替换掉；
    • 优点：减少对主存的操作到最小的程度；
